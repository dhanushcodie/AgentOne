# Contributing to AgentOne

Thanks for your interest in contributing!

## How to contribute

1. **Fork** the repository
2. **Create a branch** from `main` for your change (`git checkout -b my-improvement`)
3. **Make your changes** and commit them
4. **Open a pull request** against `main` — describe what you changed and why

All changes go through a PR. Direct pushes to `main` are not accepted.

## What makes a good PR

- One focused change per PR — easier to review and merge
- Clear PR title and description explaining what the agent does differently and why it's better

## Ideas for contributions

- Improved agent prompts (sharper questions, better synthesis, stronger critique lenses)
- New agents (e.g. a user-story generator, a technical feasibility reviewer)
- Support for new output formats
- Improvements to the Claude Code skill (`skills/agentone.md`)
- Improvements to the Python CLI (`scripts/PlanningAgent/`)

## Questions

Open a [GitHub Issue](https://github.com/dhanushcodie/AgentOne/issues) to discuss ideas before building something large.
